# Архитектура приложения

## Общая схема

Приложение состоит из трех основных слоев:

1. **UI Layer** - пользовательский интерфейс (PySide6)
2. **Core Layer** - бизнес-логика и сервисы
3. **Jira Layer** - получение задач Jira по источникам и кэширование

## Поток выполнения

### Обработка горячей клавиши

1. Пользователь нажимает глобальную горячую клавишу (по умолчанию `Ctrl+=`)
2. `pynput` перехватывает комбинацию и вызывает обработчик в `Application`
3. `Application` захватывает выделенный текст через `ClipboardService`
4. Для глобальной клавиши открывается `CommentDialog`
5. Пользователь выбирает комментарий
6. Если комментарий привязан к Jira-источнику:
   - Получаются задачи из `JiraIssuesService`
   - При единственной задаче выполняется авто-выбор
   - Исключение: если показаны данные из устаревшего кэша без успешного обновления, всегда показывается `IssueDialog`
7. `Application` восстанавливает фокус исходного окна
8. `ClipboardService` обрабатывает текст:
   - Симулирует `Ctrl+C`
   - Читает буфер
   - Применяет шаблон
   - Записывает в буфер
   - Симулирует `Ctrl+V`

## Ключевые компоненты

### MainWindow
- Управление настройками
- CRUD операции с комментариями
- Перетаскивание строк комментариев для изменения порядка (без автосохранения в файл)
- Системный трей
- Команда "Обновить источники" в меню трея

### CommentDialog
- Выбор комментария
- Поиск и фильтрация
- Модальное окно always-on-top

### IssueDialog
- Выбор задачи Jira из списка
- Поиск и фильтрация
- Кнопка `Обновить` для повторного запроса к источнику
- Обновление списка выполняется асинхронно (в фоне), UI остаётся отзывчивым
- Кнопка `OK` назначена кнопкой по умолчанию (Enter)

### HotkeyManager
- Регистрация глобальных хоткеев
- Использует `pynput`

### ClipboardService
- Симуляция Ctrl+C/V
- Чтение/запись буфера (через pyperclip)
- Применение шаблонов
- Нормализация отступов вставляемого блока
- Сохранение и восстановление исходного буфера обмена

### TemplateEngine
- Обработка макросов, модификаторов и блочных директив
- Поддерживаемые макросы:
  - `{text}` - выделенный текст
  - `{date}`, `{time}`, `{datetime}` - время
  - `{author}` - автор из настроек приложения
  - `{issue_key}`, `{issue_summary}` - Jira
- Поддерживаемые расширения:
  - `{text|prefix="..."}` - префикс каждой непустой строки `{text}`
  - `{@line_limit ...}{@end}` - ограничение длины строк внутри блока

### CommentsRepository
- Хранение комментариев в `comments.json`
- CRUD операции
- Поиск по комментариям
- Порядок комментариев хранится в том же порядке, в котором они сохранены в файле

### JiraSourcesRepository
- Хранение Jira-источников в `jira_sources.json`
- CRUD операции через таблицу в окне "Источники"
- Для источника доступны `ttl_minutes`, `timeout_seconds`, `auto_refresh`

### JiraIssuesCache
- Кэширование задач Jira по источникам
- TTL задается в источнике (минуты)
- `0` отключает кэш, `-1` хранит кэш без срока
- Хранение в `jira_issues_cache.json`

### JiraLastIssueRepository
- Хранение последней выбранной Jira-задачи по источнику в `jira_last_issue.json`
- Используется для поднятия последней выбранной задачи наверх списка в `IssueDialog`

### JiraIssuesService
- Получение задач из Jira REST API (`/rest/api/2/search`)
- Ограничение: максимум 20 задач
- Таймауты берутся из источника (секунды)
- Стратегия `stale-while-revalidate`: при просроченном кэше данные отдаются сразу, а обновление выполняется в фоне
- При отсутствии кэша выполняется live-запрос; при таймауте ошибка возвращается без повторного блокирующего запроса

### Application (автообновление Jira)
- Фоновый планировщик проверяет автообновляемые источники
- Источник обновляется только если `auto_refresh=true` и `ttl_minutes > 0`
- При неактивности пользователя более 60 минут автообновление временно приостанавливается

### Безопасная запись конфигурации
- JSON-файлы конфигурации и кэша записываются через `atomic_write_json(...)`
- Схема записи: временный файл в той же директории -> `flush`/`fsync` -> `os.replace`
- Цель: минимизировать риск повреждения файла при аварийном завершении/отключении питания

## Точки расширения

### Добавление новых макросов

Зарегистрируйте обработчик в `TemplateEngine._macro_handlers`

### Несколько хоткеев

Регистрируйте несколько комбинаций в `_setup_hotkeys()`
