# Архитектура приложения

## Общая схема

Приложение состоит из трех основных слоев:

1. **UI Layer** - пользовательский интерфейс (PySide6)
2. **Core Layer** - бизнес-логика и сервисы
3. **Jira Layer** - получение задач Jira по источникам и кэширование

## Поток выполнения

### Обработка горячей клавиши

1. Пользователь нажимает глобальную горячую клавишу (по умолчанию `Ctrl+=`)
2. `pynput` перехватывает комбинацию и вызывает обработчик в `Application`
3. `Application` захватывает выделенный текст через `ClipboardService`
4. Для глобальной клавиши открывается `CommentDialog`
5. Пользователь выбирает комментарий
6. `Application` восстанавливает фокус исходного окна
7. `ClipboardService` обрабатывает текст:
   - Симулирует `Ctrl+C`
   - Читает буфер
   - Применяет шаблон
   - Записывает в буфер
   - Симулирует `Ctrl+V`

## Ключевые компоненты

### MainWindow
- Управление настройками
- CRUD операции с комментариями
- Системный трей
- Команда "Обновить источники" в меню трея

### CommentDialog
- Выбор комментария
- Поиск и фильтрация
- Модальное окно always-on-top

### HotkeyManager
- Регистрация глобальных хоткеев
- Использует `pynput`

### ClipboardService
- Симуляция Ctrl+C/V
- Чтение/запись буфера (через pyperclip)
- Применение шаблонов
- Нормализация отступов вставляемого блока
- Сохранение и восстановление исходного буфера обмена

### TemplateEngine
- Обработка макросов, модификаторов и блочных директив
- Поддерживаемые макросы:
  - `{text}` - выделенный текст
  - `{date}`, `{time}`, `{datetime}` - время
  - `{author}` - автор из настроек приложения
  - `{issue_key}`, `{issue_summary}` - Jira
- Поддерживаемые расширения:
  - `{text|prefix="..."}` - префикс каждой непустой строки `{text}`
  - `{@line_limit ...}{@end}` - ограничение длины строк внутри блока

### CommentsRepository
- Хранение комментариев в `comments.json`
- CRUD операции
- Поиск по комментариям

### JiraSourcesRepository
- Хранение Jira-источников в `jira_sources.json`
- CRUD операции через таблицу в окне "Источники"

### JiraIssuesCache
- Кэширование задач Jira по источникам
- TTL задается в источнике (минуты)
- `0` отключает кэш, `-1` хранит кэш без срока
- Хранение в JSON

### JiraIssuesService
- Получение задач из Jira REST API (`/rest/api/2/search`)
- Ограничение: максимум 20 задач
- Таймауты берутся из источника (секунды), fallback на кэш

## Точки расширения

### Добавление новых макросов

Зарегистрируйте обработчик в `TemplateEngine._macro_handlers`

### Несколько хоткеев

Регистрируйте несколько комбинаций в `_setup_hotkeys()`
