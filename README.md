# 1C Comment Hotkeys Utility

Десктопная утилита для Windows, которая помогает разработчику быстро и единообразно комментировать код по шаблонам.

## Описание

Программа работает в фоне и автоматизирует шаблонное комментирование кода:
1. Разработчик выделяет фрагмент кода.
2. Нажимает глобальную горячую клавишу.
3. Выбирает нужный шаблон комментария.
4. Утилита вставляет готовый комментарий с подстановкой макросов (дата/время/автор/Jira) и форматированием.

Это снижает ручную рутину, ускоряет оформление изменений и делает комментарии в команде более единообразными.

## Использование

1. После запуска программа сворачивается в системный трей
2. В меню трея доступна команда `Обновить источники` для принудительного обновления Jira-кэша
3. В окне настроек можно изменить глобальный хоткей (по умолчанию `Ctrl+=`) и сохранить
4. Пустая глобальная комбинация допускается: в этом режиме работает только быстрый хоткей у шаблонов
5. Нажмите глобальный хоткей
6. В открывшемся диалоге выберите нужный комментарий:
   - Используйте поиск для фильтрации
   - Двойной клик или кнопка "OK" для выбора
   - ESC или "Отмена" для отмены
7. Выделенный текст будет автоматически модифицирован согласно шаблону

## Конфигурация

### Файлы конфигурации

Конфигурационные файлы хранятся в:
- **Windows (обычная установка Python):** `%APPDATA%\1CCommentHotkeys\`
- **Windows (Python из Microsoft Store):** `%APPDATA%\1CCommentHotkeys\` (при первом запуске выполняется перенос из legacy-пути в `LocalCache\Roaming`, если данные там уже есть)

### Файл config.json

```json
{
  "hotkeys": [
    {
      "combination": "ctrl+=",
      "action": "show_comment_dialog"
    }
  ],
  "ui": {
    "start_minimized": true,
    "log_to_file": false,
    "author": "AUTHOR"
  }
}
```

Горячая клавиша и UI-настройки читаются из `config.json`.
`log_to_file=true` включает запись логов в файл `app.log` в каталоге настроек.

### Файл comments.json

Содержит список шаблонов комментариев:

```json
[
  {
    "name": "Код добавлен",
    "template": "{@line_limit max=110 mode=wrap suffix=\"// \"}// + {author}. {datetime}. [{issue_key}] {issue_summary}{@end}\n{text}\n// - {author}. {datetime}.",
    "hotkey": "ctrl+alt+1",
    "source": "Мой Jira фильтр",
    "hidden": false
  },
  {
    "name": "Код удален",
    "template": "{@line_limit max=110 mode=wrap suffix=\"// \"}// + {author}. {datetime}. [{issue_key}] {issue_summary}{@end}\n{text|prefix=\"// \"}\n// - {author}. {datetime}."
  },
  {
    "name": "Функция добавлена",
    "template": "{@line_limit max=110 mode=wrap suffix=\"// \"}// {author}. {datetime}. [{issue_key}] {issue_summary}{@end}"
  },
  {
    "name": "Коммит",
    "template": "[{issue_key}] {issue_summary}"
  },
  {
    "name": "Метаданные",
    "template": "// {author}. {date}. {issue_key}."
  }
]
```

`hotkey` (опционально) задаёт быструю вставку конкретного комментария без диалога выбора.  
`source` (опционально) хранит название Jira-источника для комментария.
`hidden` (опционально) скрывает комментарий из диалога при нажатии глобальной горячей клавиши.

### Файл jira_sources.json

Содержит источники Jira для комментариев:

```json
[
  {
    "name": "Мой Jira фильтр",
    "url": "https://your-domain.atlassian.net/issues/?jql=project%20%3D%20PROJECT%20ORDER%20BY%20updated%20DESC",
    "token": "your-api-token",
    "ttl_minutes": 5,
    "timeout_seconds": 2
  }
]
```

Важно для подстановки задач Jira:
- Нужно создать источник в окне `Источники` (название, ссылка на Jira-фильтр, токен).
- В нужном шаблоне комментария в поле `Источник` выбрать этот источник.
- Только при таком связывании источника и шаблона в комментарий будут подставляться `{issue_key}` и `{issue_summary}`.

Требование к ссылке источника:
- Ссылка должна вести на настроенный фильтр Jira с нужными задачами.
- В URL обязательно должен присутствовать параметр `jql` (именно он используется для запроса списка задач).

Настройки источника:
- `ttl_minutes` (мин) — время жизни кэша. `0` отключает кэш, `-1` хранит кэш без срока. По умолчанию `5`.
- `timeout_seconds` (сек) — таймаут запросов к Jira. Минимум `1`. По умолчанию `2`.

## Макросы в шаблонах

Поддерживаемые макросы:
- `{text}` - выделенный текст
- `{date}` - текущая дата в формате `dd.MM.yyyy`
- `{time}` - текущее время в формате `HH:mm`
- `{datetime}` - дата и время в формате `dd.MM.yyyy HH:mm`
- `{author}` - автор из поля `Автор` в главном окне
- `{issue_key}` - ключ задачи Jira (если выбрана задача)
- `{issue_summary}` - краткое описание задачи Jira

## Расширенный синтаксис шаблонов

В шаблонах поддерживаются:
- обычные макросы (`{text}`, `{date}` и т.д.)
- модификаторы макросов (`{text|...}`)
- блочные директивы (`{@...} ... {@end}`)

### Модификатор `prefix` для `{text}`

Формат:
- `{text|prefix="..."}`

Поведение:
- Префикс добавляется к каждой **непустой** строке значения `{text}`.
- Пустые строки остаются пустыми.
- При наличии отступов префикс занимает место внутри отступа, чтобы код оставался на той же "колонке".

Пример:

```text
{text|prefix="// "}
```

Если выделенный текст:

```text
line 1

line 2
```

Результат:

```text
// line 1

// line 2
```

### Блочная директива `line_limit`

Формат:
- `{@line_limit max=110 mode=wrap suffix="// "}`
- `...текст и/или макросы внутри блока...`
- `{@end}`

Параметры:
- `max` - максимальная длина строки в символах (целое число, `<= 0` отключает ограничение).
- `mode` - режим обработки. Сейчас поддерживается `wrap` (перенос по словам).
- `suffix` - префикс, который добавляется к продолжению перенесенной строки.

Важно:
- Ограничение применяется **только к содержимому внутри блока**.
- Это позволяет, например, ограничивать длину заголовка комментария и не трогать `{text}` с кодом.
- Перенос выполняется по пробелам; длинные слова без пробелов не разрываются.

Пример выборочного применения:

```text
{@line_limit max=110 mode=wrap suffix="// "}
// + {author}. {datetime}. [{issue_key}] {issue_summary}
{@end}
{text}
// - {author}. {datetime}.
```

В этом примере:
- ограничивается только первая строка с задачей,
- `{text}` вставляется в исходном виде,
- `suffix="// "` добавляется к строкам-продолжениям при переносе.
